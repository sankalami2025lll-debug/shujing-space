# Skill：提交前代码审查

当任务是“检查代码、准备提交、修 bug、优化稳定性”时使用。

## 审查清单
1. 是否通过 `pnpm build`。
2. 是否有 TypeScript 明显错误。
3. 是否改动了无关文件。
4. 是否破坏导航和页面跳转。
5. 是否保留中文注释。
6. 是否有未处理的 loading/error/empty 状态。
7. 是否提交了密钥、Token、数据库连接串。
8. 是否引入了不必要依赖。

## 输出格式
- 结论：可提交 / 不建议提交
- 风险点
- 必须修复
- 可后续优化

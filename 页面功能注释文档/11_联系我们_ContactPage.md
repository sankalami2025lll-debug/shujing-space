# 11 联系我们 ContactPage 页面说明


> 项目：数境空间公司官网  
> 来源：Figma Make 当前链接导出源码  
> 设计依据：Figma 当前导出版本  
> 前端技术栈：React + Vite + TypeScript + Tailwind CSS + lucide-react + shadcn/ui
> 技术栈说明：当前 Vite + React 前端为 UI 原型基准（视觉/文案/交互的还原依据，不得删除）；最终生产前端将迁移至 Next.js，迁移后保持现有 UI/文案/交互一致。后端 NestJS、数据库 PostgreSQL、部署 1Panel + Docker，详见 docs/backend-architecture-plan.md。  
> 说明：本文档按页面/模块拆分，供前端、后端和 AI 编程工具直接理解页面功能、交互、状态、接口与注释位置。


## 对应源码

- `src/app/ContactPage.tsx`

## 页面定位

联系我们页面是官网最重要的业务转化页面，用于承接客户咨询、项目需求、训练数据服务申请和模型服务需求。

## 页面结构

| 区域 | 功能 |
|---|---|
| Hero | 服务咨询入口和页面定位 |
| 咨询表单 | 收集客户需求 |
| 我们可以提供 | 展示服务能力 |
| 联系方式 | 展示电话、邮箱、地址等信息 |
| 服务流程 | 需求沟通、数据评估、方案确认、交付应用 |
| 服务方向 | 展示服务业务方向 |
| CTA | 再次引导提交需求 |
| Footer | 底部信息 |

## 表单字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 姓名 | 文本输入 | 联系人姓名 |
| 手机 / 微信 | 文本输入 | 联系方式 |
| 公司名称 | 文本输入 | 企业客户信息 |
| 邮箱 | 文本输入 | 可选或必填，视业务要求 |
| 应用场景 | 下拉选择 | 工程改造、数字文旅、云上营销等 |
| 数据类型 | 多选标签 | 实景三维、BIM、构件级、具身智能训练场景等 |
| 项目阶段 | 单选标签 | 了解咨询、已有项目、需要报价、长期合作等 |
| 需求描述 | 文本域 | 客户具体需求 |
| 预算范围 | 下拉选择 | 预算区间 |

## 状态说明

| 状态 | 说明 |
|---|---|
| `submitted` | 表单是否提交成功 |
| `selectedDataTypes` | 选择的数据类型数组 |
| `selectedStage` | 当前项目阶段 |
| `selectedScene` | 当前应用场景 |
| `selectedBudget` | 当前预算范围 |

## 当前交互

- 点击「提交需求」后，当前只显示成功状态。
- 点击「重新填写」可回到表单。
- 点击 Hero 中的按钮可滚动到表单区域。
- 点击「浏览模型」可进入模型库。

## 代码注释建议

```tsx
// ContactPage：业务咨询与数据服务需求收集页，是官网核心转化页面。
// selectedDataTypes：用户选择的数据类型，可多选。
// selectedStage：项目阶段单选状态。
// submitted：当前为前端模拟提交，正式开发应根据接口返回结果显示成功或失败。
// scrollToForm：用于 Hero 和 CTA 按钮平滑滚动到表单。
```

## 后端接口建议

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/contact/leads` | POST | 提交客户咨询线索 |
| `/api/contact/options` | GET | 获取表单选项配置 |
| `/api/admin/leads` | GET | 后台查看线索列表 |
| `/api/admin/leads/:id/status` | PATCH | 后台更新线索状态 |

## 线索状态建议

| 状态 | 说明 |
|---|---|
| `new` | 新线索 |
| `contacted` | 已联系 |
| `qualified` | 有效需求 |
| `quoted` | 已报价 |
| `won` | 已成交 |
| `lost` | 未成交 |

## 验收标准

- 表单所有输入控件可正常操作。
- 多选标签有选中样式。
- 提交后显示成功提示。
- 重新填写能恢复表单。
- CTA 滚动到表单区域顺滑。

# 01 全局架构与路由说明


> 项目：数境空间公司官网  
> 来源：Figma Make 当前链接导出源码  
> 设计依据：Figma 当前导出版本  
> 前端技术栈：React + Vite + TypeScript + Tailwind CSS + lucide-react + shadcn/ui
> 技术栈说明：当前 Vite + React 前端为 UI 原型基准（视觉/文案/交互的还原依据，不得删除）；最终生产前端将迁移至 Next.js，迁移后保持现有 UI/文案/交互一致。后端 NestJS、数据库 PostgreSQL、部署 1Panel + Docker，详见 docs/backend-architecture-plan.md。  
> 说明：本文档按页面/模块拆分，供前端、后端和 AI 编程工具直接理解页面功能、交互、状态、接口与注释位置。


## 页面定位

`App.tsx` 是当前 Figma Make 原型的全站入口文件，负责：

- 管理当前页面状态。
- 组织首页内容。
- 统一控制模型社区、关于我们、模型库、联系我们、注册登录页面的跳转。
- 控制首页业务平台 / 业务场景的视频介绍弹窗。
- 向子页面传递导航回调函数。

## 当前页面状态设计

源码中使用：

```ts
const [page, setPage] = useState<"home" | "community" | "about" | "models" | "contact" | "auth">("home");
const [activeModal, setActiveModal] = useState<ModalItem | null>(null);
const [initialModelId, setInitialModelId] = useState<number | undefined>(undefined);
```

### 状态说明

| 状态 | 类型 | 作用 |
|---|---|---|
| `page` | 字符串联合类型 | 控制当前显示哪个页面 |
| `activeModal` | `ModalItem \| null` | 控制首页视频弹窗是否打开 |
| `initialModelId` | `number \| undefined` | 从社区精选模型跳转到模型库详情时指定模型 ID |

## 页面跳转关系

| 来源 | 目标 | 当前实现 |
|---|---|---|
| 首页 CTA「浏览模型」 | 模型社区 | `setPage("community")` |
| 首页 CTA「了解我们」 | 关于我们 | `setPage("about")` |
| 模型社区「浏览模型」 | 模型库 | `setPage("models")` |
| 精选模型卡片 | 模型详情 | `setInitialModelId(model.id); setPage("models")` |
| 顶部导航「联系我们」 | 联系我们 | `setPage("contact")` |
| 顶部导航「注册 / 登录」 | 登录注册页 | `setPage("auth")` |

## 正式开发建议路由

```txt
/                       首页
/community              模型社区入口页
/models                 模型库列表页
/models/:id             模型详情页
/about                  关于我们
/contact                联系我们
/auth                   注册 / 登录
/user                   个人中心
/user/models            我的模型
/user/favorites         我的收藏
/user/published         我的发布
/user/applications      我的申请
```

## 前端代码注释要求

建议在 `App.tsx` 中补充以下注释：

```ts
// 全站页面状态：Figma 原型使用 useState 模拟路由，正式项目建议替换为 react-router-dom。
// home：首页；community：模型社区入口；models：模型库；about：关于我们；contact：联系我们；auth：注册登录。

// 首页弹窗状态：点击业务平台或业务场景卡片后，打开对应的视频介绍弹窗。

// 模型详情入口：从精选模型卡片跳转时，先记录 initialModelId，再切换到模型库页面。
```

## 后端对接重点

全局层不直接请求后端，但需要定义：

- 用户登录状态接口：判断导航栏是否显示「注册 / 登录」或用户头像。
- 权限状态接口：判断是否允许发布模型、收藏模型、申请训练数据。
- 全局站点配置接口：Logo、Footer、联系方式、备案信息可由后台维护。

## AI 开发注意事项

- 不要把所有页面长期写在 `App.tsx` 中，应拆分为路由页面。
- 不要删除 Figma 已有页面和交互，只能规范化和工程化。
- 所有页面保持当前黑白灰高级科技风，只允许少量冰蓝和紫色作为高亮。

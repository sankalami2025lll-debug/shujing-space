# 14 给 AI 编程工具的开发执行提示词


> 项目：数境空间公司官网  
> 来源：Figma Make 当前链接导出源码  
> 设计依据：Figma 当前导出版本  
> 前端技术栈：React + Vite + TypeScript + Tailwind CSS + lucide-react + shadcn/ui
> 技术栈说明：当前 Vite + React 前端为 UI 原型基准（视觉/文案/交互的还原依据，不得删除）；最终生产前端将迁移至 Next.js，迁移后保持现有 UI/文案/交互一致。后端 NestJS、数据库 PostgreSQL、部署 1Panel + Docker，详见 docs/backend-architecture-plan.md。  
> 说明：本文档按页面/模块拆分，供前端、后端和 AI 编程工具直接理解页面功能、交互、状态、接口与注释位置。


## 总体提示词

你正在开发「数境空间」官网前端与后端。请严格基于 Figma Make 当前导出版本 的页面结构、视觉风格和交互逻辑实现，不要自行改变业务结构。

技术栈建议：

- 前端：React + Vite + TypeScript + Tailwind CSS
- UI：保留当前 shadcn/ui 和 lucide-react 图标体系
- 路由：react-router-dom
- 状态：React hooks，后续可接 Zustand 或 Redux Toolkit
- 后端：Node.js / NestJS / Express 均可
- 数据库：PostgreSQL 或 MySQL
- 文件存储：S3 / OSS / COS / 本地 MinIO

## 必须保留的页面

- 首页 `/`
- 模型社区入口 `/community`
- 模型库 `/models`
- 模型详情 `/models/:id`
- 关于我们 `/about`
- 联系我们 `/contact`
- 注册登录 `/auth`
- 个人中心 `/user`

## 必须保留的核心交互

- 首页业务平台卡片点击打开视频介绍弹窗。
- 首页业务场景卡片点击打开视频介绍弹窗。
- 模型社区精选模型点击进入对应模型详情。
- 模型库支持搜索、分类筛选、排序。
- 模型卡片支持浏览、点赞、收藏、分享。
- 用户可以发布模型。
- 只有具身智能机器人训练场景显示训练数据服务申请入口。
- 联系页面表单提交后显示成功状态。
- 登录注册页面支持 Tab 切换、验证码倒计时、密码显示切换。

## 代码注释要求

开发时必须在每个页面顶部加入页面级注释：

```tsx
/**
 * 页面名称：xxx
 * 页面功能：说明该页面承担的业务目标
 * 主要模块：列出页面内主要 UI 区块
 * 主要交互：列出按钮、弹窗、表单、跳转逻辑
 * 后端接口：列出需要对接的接口
 */
```

每个关键状态必须添加注释：

```tsx
// 当前模型分类筛选条件
const [activeType, setActiveType] = useState("全部模型");
```

每个提交函数必须添加注释：

```tsx
// 提交训练数据申请：正式环境中需要调用 POST /api/training-applications
const handleSubmit = async () => {{}}
```

## 禁止事项

- 不要把模型社区页面写成公司介绍页。
- 不要把「申请数据服务」扩展成多个无关类型，目前只有具身智能机器人训练场景这一类。
- 不要删除当前黑白灰高级科技视觉风格。
- 不要使用廉价大面积蓝色渐变。
- 不要把所有页面继续塞在 `App.tsx`，必须拆分路由。
- 不要忽略移动端适配。

## 开发顺序

1. 初始化 React + Vite + TypeScript + Tailwind 项目。
2. 建立路由结构。
3. 抽离 `NavBar`、`Footer`、`ModelCard`、`Modal` 等公共组件。
4. 实现首页、关于我们、联系我们。
5. 实现模型社区和模型库。
6. 实现模型详情、上传弹窗、训练申请弹窗、个人中心。
7. 接入后端接口。
8. 增加表单校验、鉴权、错误提示、加载状态。
9. 完成移动端和性能优化。
10. 完成测试和上线。

## 验收总标准

- 页面视觉与 Figma 当前导出版本 保持一致。
- 每个页面都有清晰注释。
- 每个功能都有明确交互反馈。
- 每个表单都有校验和提交结果。
- 所有动态数据都能从后端接口读取。
- 未登录权限和登录后权限区分明确。
